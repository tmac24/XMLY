商品档案

  • 按照功能模式来分

    • 新增模式
    • 编辑模式
    • 查看模式

    其中，新增模式，编辑模式用的是同一套UI风格，查看模式是另一种UI风格

商品档案包括以下属性 接下来，根据业务的复杂性逐一说明

基本单位

商品单位的标识，只有一个单位。比如：1个，3只等

辅助单位

有时候商品只有基本单位是不够的，比如：酒1箱5瓶 这时候需要辅助单位。辅助单位也是sku2，属于规格级。后面商品规格会讲到。

商品规格

本系统的商品规格有5级

  • color
  • size
  • cup
  • sku1
  • sku2

color,size是有云端库的，其他都没有

skuStr：服务器返回的字段,表示规格级。用于商品规格，库存预警，价格管理。

例如：1001_ 1003_ 1006_ 1012_ 1056 注意 此顺序不可乱

  • 1001表示color
  • 1003表示size
  • 1006表示cup
  • 1012表示sku1
  • 1056表示sku2

微商城设置

此模块用于设置网页商城

  • 商品分享说明
  • 微商城图片

价格管理

依赖于商品规格，价格类型根据本地数据库表查询

  • 基本价格（未开启多规格：无规格单单位）
  • 统一定价（已开启多规格：无规格多单位）
  • 特殊定价（已开启多规格：多规格多单位/多规格单单位）

库存

设置本地仓库的成本价和库存数量

库存预警

依赖于商品规格 设置本地仓库的库存上限和下限

条形码

  • 商品条形码 （普通商品级别）
  • 规格条形码（商品的规格级别）

属性

用于定义商品的属性，例如季节性的商品

  • 单选
  • 数值
  • 文本
  • 日期
  • 多选

从项目工程了解商品档案

整个商品档案分为四个部分,文件还是很多的

本地数据库

由于本地是有数据库的，商品的一些基本信息都存在数据库中，用的时候需要我们自己查询本地数据库，本地数据库是我们登陆软件之后就会下载到本地，供我们增删改查。

以下是一些常用的表

#pragma mark - 本地数据库查询

//获取供应商
- (void)getSupplyArray {
    YYDBManager * db = [YYDBManager sharedManager];
    _supplyArray = [NSMutableArray arrayWithArray:[db getSupply_noStop:YES]];
}

//获取价格
- (void)getPriceArray {
    YYDBManager * db = [YYDBManager sharedManager];
    _priceTypeArray = [NSMutableArray arrayWithArray:[db getPriceType]];
}

//获取单位
- (void)getUnitArray {
    YYDBManager * dbManger = [YYDBManager sharedManager];
    _unitArray = [NSMutableArray arrayWithArray:[dbManger getUnit]];
}

//获取仓库
- (void)getStoreArray {
    _storeArray = [NSMutableArray array];
    YYDBManager * db = [YYDBManager sharedManager];
    NSArray * temp = [db getStore_noStop:YES];
    for (storeModel * d in temp)
    {
        XQWareHouseModel * model = [XQWareHouseModel new];
        model.StoreID = (int)d.storeid;
        model.StoreName = d.storename;
        [self.storeArray addObject:model];
    }
}

新增、编辑

商品新增和商品编辑用的都是一套代码，界面风格都是一样的

  • ISManualAddGoodsPresenter用于处理数据,包括数据库的查询帮助控制器减轻压力，
  • 服务返回的数据都是id，需要通过id查询本地数据库，获取对应的名称等
  • BindCell中的文件是cell的类扩展，用来绑定数据
  • 二级界面跳转含有原生和H5，以下二级界面为h5
    • 基本单位，辅助单位
    • 统一定价
    • 默认供应商
    • 商品规格设置
    • 商品属性设置

核心代码 通过字典来绑定数据，包括界面的展示，界面的跳转

/// 返回一个字典
/// @param title 标题
/// @param subTitle 副标题
/// @param cellType 标识符
/// @param skipID 跳转标识符
- (NSDictionary *)dictWithTitle:(NSString *)title subTitle:(id)subTitle cellType:(ISLookRecordCellType)cellType skipID:(id )skipID{
    return @{@"title":title, @"subTitle":subTitle, @"cellType": @(cellType) ,@"skipID":skipID};
}

数据结构

商品规格

  • 选中规格的数据是很重要的，价格管理，库存，库存预警，都是依赖于它
  • 需要查询本地数据库对应的规格级组，再通过id找到对应规格的名称

库存预警和价格管理的数据通过SKUIDStr是交互在一起的

第一个0代表color 第二个0代表size 第三个0代表cup 第四个0代表sku1 第五个0代表sku2

注意

  • 全部都是0代表无规格单单位的商品，有值就是其他种情况了，前面商品规格讲过
  • SKUIDStr用来标识是哪种规格级别的商品
  • StoreID标识哪种仓库，仓库名需要查询本地数据库
  • PriceID标识哪种价格类型，同样需查询本地数据库
  • 服务器拉取和上传的数据都是这种数据结构，上传数据需要双重遍历数据（价格管理、库存预警）
